# Task: Conversation Summary Generation and Persistence

## Description
Implement `generate_and_save_summary()` in `store_assistant/agent/agent.py` (or `store_assistant/agent/summary.py`). This function is called when a session terminates — it uses the LLM to generate a rich narrative summary and persists it to the `conversation_summaries` table with full metadata.

## Background
When a session ends (farewell or 3 off-topic strikes), the system must generate a human-readable summary of the conversation and save it to SQLite. The summary is generated by calling `ChatOpenAI` directly (not through the tool-calling agent), using the conversation history and session metadata as context. The summary text plus metadata is then saved via `db.save_summary()`.

- Design: `.agents/planning/store_assist/design/detailed-design.md`

## Technical Requirements
1. Function signature: `generate_and_save_summary(history: list[dict], state: SessionState) -> str`
2. Build a prompt that includes: all conversation turns formatted as text, `stores_saved` count, `stores_retrieved` count, instruction to write a concise 2-3 sentence narrative summary
3. Call `ChatOpenAI(model="gpt-4o-mini").invoke([HumanMessage(content=summary_prompt)])` directly
4. Extract summary text from the response
5. Compute `session_end = datetime.utcnow()`
6. Call `db.save_summary(state.session_id, summary_text, state.stores_saved, state.stores_retrieved, state.session_start, session_end)`
7. Return the summary text so it can be appended to the farewell message shown to the user
8. Integrate: call `generate_and_save_summary()` inside `chat()` immediately after `state.terminated` is set to `True`

## Dependencies
- Step 2 (`db.save_summary`) must be complete
- Step 5 (agent core, `ChatOpenAI` instance) must be complete
- Step 6 (`SessionState`) must be complete

## Implementation Approach
1. Format history as: `"\n".join([f"{m['role'].upper()}: {m['content']}" for m in history])`
2. Build summary prompt string with history, metadata, and instruction
3. Call the LLM and extract `.content` from the response
4. Call `db.save_summary()` with all required args
5. In `chat()`: after state update detects termination, call `generate_and_save_summary()` and append summary to the final response text
6. In tests: mock both the LLM call and `db.save_summary`

## Acceptance Criteria

1. **Summary LLM Is Called On Termination**
   - Given a session terminated by farewell
   - When `chat()` processes the terminating message
   - Then `ChatOpenAI.invoke` is called with a prompt containing the conversation history

2. **db.save_summary Is Called With Correct Args**
   - Given a session with `stores_saved=2`, `stores_retrieved=1`
   - When summary generation runs
   - Then `db.save_summary` is called with `stores_saved=2`, `stores_retrieved=1`, valid `session_id`, `session_start`, `session_end`

3. **Summary Text Appears in Final Response**
   - Given a farewell termination
   - When `chat()` returns
   - Then the returned response string contains the LLM-generated summary text

4. **Summary Record Persists to DB**
   - Given a real (temp) SQLite DB and a mocked LLM returning "Test summary."
   - When a farewell is triggered
   - Then querying `conversation_summaries` returns a record with `summary_text = "Test summary."`

5. **Duration Is Positive**
   - Given `session_start` was set before `session_end`
   - When the summary is saved
   - Then `duration_seconds > 0` in the DB record
